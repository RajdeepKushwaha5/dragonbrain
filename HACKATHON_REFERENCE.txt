================================================================================
THEME: POST-TRANSFORMER FRONTIER AI CHALLENGE
================================================================================

CORE PROBLEM: Transformer limitations
1. Context scales quadratically O(T²) — breaks on long documents
2. Static weights — can't learn post-training without fine-tuning
3. Memory via external DBs (vector stores, RAG) — searching, not remembering

SOLUTION: BDH (Dragon Hatchling) Architecture
- Replaces dense matrices with scale-free graph of neurons
- Sparse activations: ~5% neurons fire per token (vs ~100% in transformers)
- Hebbian synapses: constant-size O(n×d) memory regardless of context length
- O(T) linear attention complexity (vs O(T²) transformer)
- Inference-time learning: synapses strengthen via co-activation, no backprop needed
- Composable model merging: concatenate independently trained models
- Monosemantic synapses: individual synapses encode specific concepts (interpretable)
- Graph topology is directly visualizable

PROPERTY COMPARISON
| Property        | Transformer              | BDH                              |
|-----------------|--------------------------|----------------------------------|
| Structure       | Dense matrix layers      | Scale-free graph of neurons      |
| Activation      | ~100% neurons fire       | ~5% neurons fire (sparse)        |
| Memory          | KV-cache (grows w/ ctx)  | Hebbian synapses (constant size) |
| Attention       | O(T²)                   | O(T) linear                      |
| Interpretability| Black box                | Graph structure visualizable     |
